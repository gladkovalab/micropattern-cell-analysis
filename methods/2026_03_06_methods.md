# Methods – Micropattern Cell Analysis
*2026-03-06*

**Image Acquisition & Loading**
- Fluorescence microscopy images were acquired in ND2 format and loaded using the `nd2` library (v0.10.3) with xarray (v2025.7.1) for labeled multidimensional arrays.

**Micropattern Template Preparation**
- A fibronectin micropattern template was defined as an SVG file and rasterized to 1326 × 1326 pixels using CairoSVG (v2.8.2) and PyMuPDF (v1.26.4). The template was zero-padded to 2048 × 2048 pixels for frequency-domain matching.

**Template Matching (Pattern Localization)**
- The micropattern center was localized in each image by normalized cross-correlation in frequency space using 2D FFTs (NumPy v2.3.3). The 640 nm channel z-sum projection was used as the reference signal. A template matching score (fractional overlap with Otsu-thresholded image) was computed for quality control.

**Image Cropping**
- A 1024 × 1024 pixel region centered on the matched template was extracted from the z-sum projection for each fluorescence channel and saved as NetCDF4 (v1.7.3) files.

**Nuclear Segmentation**
- The nucleus was segmented from the 405 nm (DAPI) channel sum projection using Otsu thresholding (scikit-image v0.25.2). The largest connected component was selected, and a Euclidean distance transform (SciPy v1.16.2) was computed from the nuclear boundary.

**Background Subtraction**
- Mitochondrial signal (488 nm channel) background was estimated from the 99.99th percentile of the leftmost and rightmost 128-pixel column strips of the cropped image and subtracted.

**Perinuclear & Peripheral Zone Quantification**
- Euclidean distance transforms were computed from the nuclear boundary and from the top arch of the micropattern contour. Perinuclear (< 5 µm from nucleus) and peripheral (< 5 µm from arch, outside perinuclear) masks were defined. Mitochondrial signal was summed within each mask and expressed as a percentage of total cropped signal.

**Data Output**
- Per-cell metrics were compiled into CSV and Excel files using Polars (v1.36.1) and XlsxWriter. Diagnostic figures were saved as multi-page PDFs using Matplotlib (v3.10.6).

**Prototyping & Visualization**
- Analysis pipelines were prototyped and results visualized interactively using Marimo (v0.17.8), a reactive Python notebook environment.

---

All analysis was performed in Python 3.12 using packages managed via the Pixi environment manager.
